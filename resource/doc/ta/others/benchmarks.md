## சுமையில் இருக்கும் மாற்றங்கள் யாவைப்படும்?

* சுமையை சேவையகத்துக்கு இணைக்கும் மென்பொருளின் பிரமையான உலகத்தில் மாற்றூக்கம் (உள்ளே பின்னர் அல்லது அடைவதற்கு உதவியும்)
* சுமையை சேவையகத்திற்கு இணைக்கும் நிறுவல் (உள்ளே பின்னர் அல்லது அடைவதற்கு உதவியும்)
* HTTP keep-alive ஐ எப்படி இயக்கவும் (இயற்க்கைப் பாதையில் இருந்து இயக்கம் செய்யப்படும்)
* செயலழிப்பின் அண்மையில் போதுமான அலையைப் பெற மிகவும் சாதாரணமான எண்ணிக்கையைப் பெற (வெளியில் சுமையைப்பற்றி மிகுந்த எண்ணிக்கையைப் பெற முடியாது)
* சேவையக செயலாளர் பிராசசின்கள் பொதுவாகவே பொருந்தக்கூடியது (உலகத்தின் ஹெல்லோவோர்ல்ட் வேலைப்பாடுகளின் எண்ணிக்கைக்குரியதாக ஆனது, தரவு தளத்தின் வேலைப்பாடுகளின் எண்ணிக்கைக்குரியதாக ஆனது)
* நிறைவு மேல்நிலையைப் பெற மிகுந்த வண்ணம் தளத்தின் மேலே அதிகரிக்கின்றது (உலகத்தின் குறைவான TCP அண்மைகள் மூலம் ஒரு சேவையில் பதில் அனுப்பப்படும்)
HTTP Keep-Alive என்பது அதே திருத்தப்பட்ட தரவு சந்திப்பை கைவேலை செய்பதன் மூலம் பல HTTP கோரிக்கைகள் மற்றும் பதில்களைத் தமிழில் பரிசீலித்துக் கொடுக்கலாம், இது உற்பத்தி செய்யும் மென்பொருளையும் குறைக்கலாம், keep-alive ஐ முடக்கின்றால் விரித்துப் பார்ப்பதன் மூலம் சுமை விரலாவியத்தின் சுமையினையும் அதன் முன் செலவிடலாம்.
தற்போது உலாவியை ஒரு வெளிப்புற HTTP கோரிக்கையை மட்டுமல்லாமல் keep-alive ஐ ஆக்குகின்றது, அது ஒரு உலாவியை ஏற்பார்க்கும்போது மற்றும் அடுத்துவரும் கோரிக்கையை அதன் மீது மீண்டும் பயன்படுத்த வேண்டும் என்பதை அறிவிக்கும் மூலமும் சுமையத்தின் மூலமும் நேர்வழக்கத்தை ஏற்படுத்துகின்றது.
சுமைய் விருத்தியாக்கும்போது keep-alive ஐ ஆக்கவும் என்று பரிந்துரைக்கப்படுகின்றது.

## சுமையின் QPS எப்படி குறைவாகிறது?

QPS (ஒரு விசைக் குறிப்பு முறைபாடு செய்யப்படுகிறது) குறையும் புள்ளிவிவரம் கூடுதல் நேர்ந்து தொகுத்ததால் நிறைவாகும். இது அரசியலாக ஏப்படும். உதவியும், எல்லா வழிகாட்டுக்களையும் ஒன்றாய் சேர்க்கலாம். இதுவே உலகத்தில் ஆன்லைன் மேல் திரை அப்பில் முதல் QPS ஏற்படும் போது நிறைவாகும்.

மேலே பரிந்துரைக்கும்போது, ஆவனால், சுமையை சேர்க்கும்போது, சுமையின் அதிக நடவடிக்கைகளை செய்ய முடியும் (ஒரு நிற இணைப்பை ஒரு இணைப்பில் பயன்படுத்தி முடியும்). வெளியேற்றத்தை வேண்டும் போது, மேலேக்குறியதாக அளந்தபோது எண்ணிக்கையை அதிகரித்ததல் எட்டுமுகமாக வெளிவருகின்றது (இரண்டு விரிவுகளை ஒன்றில் சேர்த்துக் குறைப்பது அதனால் ஏற்படும்).
Nginx மின் பிரதி சுமையைச் சேர்க்கலாம். பின்கூடப்பட்டது. ஆனால், Nginxம் மொழிபெயர்ப்பு பிரச்சினை, மேலும் Nginx மற்றும் Webman இடையே சோதனையைச் செய்ய இருக்கின்றதும் செயல்பாடு நேரம் பயன்படுத்த வேண்டும். உள்ளே போலீவு வேட்டை என்பதால், HTTPS, SSL/TLS கையெழுத்தின் பாதுகாப்பு, தரவு மோசடி மற்றும் கிளரிக்குரிய பிடிப்புகள் கொண்டு குறைந்துவிடுகின்றன. பெறக்குறியியாக வெளிப்படுவது ஏவ்வாறு என்னவென்றால், தடமையான SSL/TLS கையெழுத்தால் ஒரு முறை உட்செதுக்கப்படும் மற்றும் HTTP கீபு-வைக்கும் (keep-alive) முறை ஆரம்பிப்பது நல்லது.

https-ஐ http-யுடன் ஒப்பிடுகின்றபோது, https-ம் SSL/TLS கையெழுத்தத்தை செய்ய, தரவு மற்றும் தரவு முறை கூடுதலாகவும், இருத்தில் கூடிய அஞ்சல் மற்றும் பயன்பாடு C மூலம் ஏதுமான செயல்முறைகள் வேகத்தை குறைக்கும். அறியப்படுகின்ற தடுக்கப்படுகின்ற ஒவ்வொன்றையும் QPS-ஐவிட குறைக்க என்ன காரணங்கள், பொதுவாக போதுமான SSL/TLS கையெழுத்தம் மற்றும் QPS-ஐ ஓராண்டு அளவிற்கு பொதுவாக கட்டுப்படுத்துகின்றன. உதவியாக ஹிட் யேவ் QPS, HTTP கீப்-அஃகை உள்ளூர் HTTP விண்ணப்பில் முதலில் செயல்படும் பொதுவான நடவடிக்கைகளைச் செய்து பாத்திக் கொண்டால் முடியும்.
 
சேப்பதையாக, சாதாரணமாக செய்திகள் 100% வாய்ந் திருப்புவேண்டும் என்பது பொதுவாக இருக்கின்றது. இரண்டு வரி முறையாகவும் கண்டிப்பாக CPU புல் உள்ளது, அதைக் கூறிலுள்ளது அதைப் பொதுவாக 100% சாகுபடியாகும். CPU-யில் திரும்ப உள்ளமைவு உள்ளது, புதிய வன்முறையில் பலமுறை செயல்பாடத்தை சேர்க்கலாம் மற்றும் QPS-ஐ உயர்த்தலாம். QPS ஐ உயர்த்துவதற்குப் பின் அனைத்து பட்டியல்களை என்ன உள்ளனவாக பார்க்க முடியும் (ஆகவே வண்ணம் அதிகமாக இருக்க முடியும்). உதாரணமாக, எந்தப் பிரிவில் சுமையைச் சேர்க்கின்றது என்பதை மின்னஞ்சல் மூலத் தளத்தில் சேர்க்கின்றது சுமையின் அதிக நடவடிக்கைகளை செய்ய முடியும்).
 
சுமையுடைய QPS ஐ அதிகரித்து செய்ய வேண்டிய வினாவுக்கு, மிகுந்த சிந்தனையுடன் விளைவுகளை கொண்டுகொள்ள முடியும் பொதுவாக, மிகுந்த சிந்தனையும் செய்யப்படவேண்டும். அதுவே பல்வேறுபரிமாண மின்னங்களைப் பயன்படுத்துகின்ற techempowerஐப் போல், webman உடன் go வடிவத்தின் இரண்ட்சு முன்னிலைகள் ஒருபோதும் ஏவவிட்டனவு அமைக்கின்றனவாக கண்டிப்பின்கவுடன் செயல்படுகின்றது. சரியான PDO மற்றும் குடிமையான SQL ஐப் பயன்படுத்தும் வழியில் webmanஐப் பயன்படுத்த முயற்சிக்கலாம்.
apipost-ன் ஒரு பிழை வகையான நீண்ட சேமித்தல் முடிவத்தை பெருத்தப்படுத்தி உள்ளது, மேலும் சேவையகம் gzip தலையை மிகுதியாக்காமல் keep-alive மையை செய்வதன் மூலம் பரிந்துரையாகும். 
பிற்படங்கள் தரக்குனரின் விரைவில் இயற்கையாக ஏரில்லாததால், இது ஒரு சேமித்தலை இலக்கு முறைக்கு பலப்படுவதாக காட்டப்படுகிறது. 
நீங்கள் மேலெழுதிய முறையில், தரக்குனருக்குப் பின்பு வரும் வார்த்தையைச் சதவீதமாக மீளப்படுத்தினால், உத்தரவாத கட்டளை 'hello webman' ஐ gzencode பயன்படுத்தி, 'Content-Encoding' என்பது 'gzip' மற்றும் உள்ள தலையை சேர்க்கும் படியில் மீளப்படுத்தப்பட வேண்டும்.

வாருங்கள், apipost ஒருவரையும் இருந்தால், அதைப் பயன்படுத்துவதில் ab இல் எடுக்கலாம் அல்லது wrk வேலைக்காட்சி பயன்படுத்துகின்றன, அபிபோஸ்டை பயன்படுத்துவதில் நன்மை இல்லை.
## வெப்மான் கட்டமைப்பாளரின் சேவைகளை முழுமையாக செயல்படுத்துங்கள்
webman ஒரு பொறியாளரின் உயர்ந்த மற்றும் மேம்பாட்டு PHP உடைய கட்டமை. அதில் CPU * 4 தன்மையானது. உண்மையில் டிராபிக் I / O -ஏன் கையெழுத்துகளின் சேவைகளில் குறியின்முறையின் உத்தரவாதத்தை அதிகரிக்க பற்றியதே crash என ஒரு சிக்கல் உண்டாகிவிடும்.
தானியங்கி, முன்னோடி கையெழுத்துகள் ஒருவரிடத்தில் நிறைவேர் செய்வதற்கான சேவைகளில், உயர்ந்த பாக்கெட் எடுப்பை அடையப்படணும்; சேவையக சேவை மாற்றுவதற்கு அளவுக்குறியீடை மாற்ற முடியும்.
# Webman உற்பத்தி நெறிப்படி

Webman ஒரு பெரிய மான உள்ள வெப் சேவை அணியில் இயக்கப்பட்டது Workerman பேஸ்ட் ஆன அசைவு PHP பிரேம்வர்க் படிவங்களை பயன்படுத்தி உள்ளது. கீழே சமர்பிக்கப்பட்டது.

## அமைப்பு சிவப்பு

```php
use Webman\App;
use support\bootstrap\Container;
use support\bootstrap\Log;
use support\bootstrap\LoadConfiguration;
use support\bootstrap\HandleExceptions;
use support\bootstrap\RegisterFacades;
use Webman\ServiceProvider as SuperServiceProvider;

require __DIR__.'/../vendor/autoload.php';

define('ROOT_PATH', __DIR__.'/..');

$providers = [
    SuperServiceProvider::class,
    Log::class,
    LoadConfiguration::class,
    HandleExceptions::class,
];

$app = new App($providers, ROOT_PATH);

Container::set('app', $app);

return $app;
```

## பொலுசுண்டி

```php
use Webman\Route;

Route::get('/', 'App\Controller\HomeController@index');
```

## மற்றுமொரு பொலுசுண்டி

```php
namespace App\Controller;

class HomeController {
    public function index() {
        return 'Hello, World!';
    }
}
```

## மென்பொருள் கண்ணாடி முன்ஸை

`factory` ஐ பதிவு செய்க.

```php
$factory->define(App\Model\User::class, function (Faker $faker) {
    return [
        'name' => $faker->name,
        'email' => $faker->unique()->safeEmail,
        'password' => bcrypt('password'),
    ];
});
```
